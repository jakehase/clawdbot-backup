## SSH Access

### Home Assistant
- **Host:** 10.0.0.7
- **Port:** 22
- **Key:** `~/.ssh/ha_esphome`
- **ESPHome configs:** `/config/esphome/`
- **Command:** `ssh -i ~/.ssh/ha_esphome root@10.0.0.7`

## Home Assistant (MCP)
- Connected via ha-mcp at 10.0.0.7:8123
- Use `mcporter call homeassistant.HomeAssistant` for API access

## Smart Home Entities
- **Hallway lights:** `light.hallway_2` (Hue group with 3 ambiance lamps)
- **Kitchen lights:** `light.kitchen`
- **Living room lamp:** `switch.living_room_lamp` (it's a switch, not a light!)
- **Presence sensors:** Aqara FP2 (indoor lux affected by room lights)

## ESP32 Devices
- **ESP32-S3-Box-3:** Voice assistant with touchscreen
  - Lights page: 2 big bulbs (hallway yellow, all lights cyan)
  - Config: `/config/esphome/esp32-s3box-3.yaml`

## Skills Needing API Keys
- **tavily:** `clawdbot config set skills.entries.tavily.apiKey "tvly_xxx"`
- **browser-use:** `clawdbot config set skills.entries.browser-use.apiKey "bu_xxx"`

## Google/Gemini Provider Config
Google models use `openai-completions` API (not google-genai), with:
- `"api": "openai-completions"`
- `"baseUrl": "https://generativelanguage.googleapis.com"`

## Why Separate?

Skills are shared. Your setup is yours. Keeping them apart means you can update skills without losing your notes, and share skills without leaking your infrastructure.
